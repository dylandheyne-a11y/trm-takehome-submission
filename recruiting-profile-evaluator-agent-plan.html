Recruiting Profile Evaluator Agent Plan

Status: future reusable skill / agent plan

Source memory: docs/RECRUITING_AGENT_MEMORY.md

Current use case: TRM Labs Senior Full-Stack Product Engineering take-home

Pithy Thesis

The agent should not just score profiles. It should help Dylan make faster recruiter decisions from thin sourcing data.

Score is the scaffolding. Verdict, evidence, archetype, and calibration are the actual judgment layer.

What This Agent Does

The agent reviews pasted LinkedIn/profile/resume notes and returns:

  1. Verdict
  2. One-line why
  3. Score
  4. Candidate archetype
  5. What we can reasonably infer
  6. What still needs validation
  7. False-positive risk
  8. Hireability read
  9. Recommended next step
  10. Probe question or outreach hook

Sourcing-Native Caveat

For sourcing, LinkedIn data is often thin. Do not treat missing detail as proof of weakness when the company, team, and archetype are strong.

If the candidate is at the right company, on the right kind of team, with the right profile archetype, the default should usually be:

> Soft Flag / validate with probe

not:

> Skip / insufficient data

This differs from full resume review. In resume review, missing evidence is more concerning because the candidate had the opportunity to present detail. In sourcing, missing evidence is normal.

Verdict Taxonomy

Use these consistently:

VerdictMeaningAction
Strong YesClear fit from visible evidencePrioritize for slate or outreach
YesGood fit with manageable unknownsAdvance, validate unknowns
Soft FlagPromising archetype but limited profile evidenceKeep in longlist; use probe question
NoMissing core functional fitPass and log reason
Def NoActive disqualifierPass immediately
SkipToo little data and weak company/team/archetype priorSeek more info only if needed

TRM-Specific Candidate Archetypes

Prefer candidates who fall into one of these archetypes:

  1. API / Developer Platform Builder
  2. Full-Stack Product Systems Owner
  3. Security / Compliance Workflow Engineer
  4. Enterprise Workflow / Internal Tools Builder
  5. Data / Search / Knowledge Systems Engineer
  6. Billing / Metering / Usage Platform Engineer
  7. Product Platform / Shared Capabilities Engineer
  8. Reliability-minded Product Engineer
  9. Applied AI Product Engineer with customer workflow depth

TRM False Positive Risks

Log and watch for:

  • API consumer, not API builder
  • Frontend-heavy full-stack profile with no backend/API ownership
  • Infra/platform profile adjacent to product but not building product systems
  • AI-company profile with hype signal but no product/backend evidence
  • Security/compliance-adjacent profile with no product engineering ownership
  • Strong company but early-career / insufficient senior scope
  • Job hopper pattern with no 2+ year compounding impact

Opacity Rule For Sourcing

If LinkedIn is thin but the candidate has:

  • strong company signal,
  • relevant team/function signal,
  • plausible seniority,
  • US-based location,
  • and no obvious disqualifier,

then do not over-penalize. Mark:

> Soft Flag: strong archetype, needs validation

Then generate a probe question.

Example probe:

> Your work at Vanta caught my eye, especially the compliance/platform side. I am curious whether you have owned backend/API systems or customer-facing workflow products end to end.
